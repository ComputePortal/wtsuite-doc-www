CSS documentation

1. Getting started

Getting started

Style sheets can be defined using conventional style tags in the head block, or via the style attribute, which wtsuite tries to intelligently turn into a common site-wide stylesheet.

style tag

Linking to a separate style sheet file is done with the link tag:

head
link(href="/style.css" rel="stylesheet" type="text/css")

Style sheet markup can be entered directly in the style tag:

head
style
body {
height: 100%;
}

Named styles

Named styles are like variable directives:

export style main
body {
height: 100%;
}

Named styles can be transpiled into css files by wt-style:

> wt-style main.wtt

main.wtt in this case must export a style variable or dictionary variable called main.


Named styles can also be used directly in style tags:

import {main} from "./main.wtt"
head
style
$main

CSS extensions

Like LESS and SCSS wtsuite provides some extensions of css to make it much easier to build complex websites with.


The following example code demonstrates many of these features:

var mobileMedia = "@media screen and (max-width: 700px)"

style main
section {
margin: 4rem $(m := 20%) 4rem $m;
width: $(100% - 2*$m);

.quotes-wrapper {
display: grid;
grid-template-columns: 1fr 1fr;
grid-column-gap: 10%;
grid-row-gap: 4rem;

$mobileMedia {
display: block;
> * {
margin-bottom: 4rem;
:last-child {
margin-bottom: 0;
}
}
}
}
}

Note that both selectors and rules accept substituted variables. Also note the bubbling of nested rules.

@wrap

To avoid div-soup, and to keep your html markup free from any styling related concerns, wtsuite has a @wrap css extension.


It allows you to wrap any tag in your html with a div, or another tag-type, from within your stylesheet:

style
@wrap section {
class: section-wrapper; // creates a
wrapper
}

The @wrap rules are evaluated before the any other rules in your stylesheet. They must be declared at the top-level of the stylesheet, and are evaluated in the order they are encountered by the parser (allow you to nest wrappers).

@wrap-siblings

@wrap-siblings is similar to @wrap, but wraps neighbouring siblings in the same wrapper tag.
