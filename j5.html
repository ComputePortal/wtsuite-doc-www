Type guards

instanceof

Like typescript, the instanceof operator can be used as a type guard condition in combination with an if statement:

function toNumber(x any) Number {
if (x instanceof String) {
return Number.parseFloat(x);
} else if (x instanceof Number) {
return x;
} else if (x instanceof Boolean) {
return x ? 1 : 0;
} else if (x instanceof Array) {
return x.length;
} else {
throw new Error("Error: can't be converted into a number");
}
}

console.log(toNumber("0.0")); // 0.0
console.log(toNumber(0.0)); // 0.0
console.log(toNumber(false)); // 0.0
console.log(toNumber([])); // 0.0
console.log(toNumber({})); // Error: can't be converted into a number

Details

Type guards only work if the left-hand side of instanceof is a simple variable, and the right-hand side is a class or interface. The type of the variable is altered for the scope of the type guard branch.


Type guard conditions can be placed inside parentheses, and can be combined with other type guard conditions using the && operator:

if ((x instanceof Number) && (y instanceof Number)) {
console.log("both x and y are numbers in this scope");
}
