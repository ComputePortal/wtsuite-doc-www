Type parameters

Some built-in types require parameters to be completely typed. Similarly to java and c++, angled brackets are used for type parameters.

Array

An array takes a single optional type parameter, which can itself be an array:

var x Array<Array<Number>> = [[0.0, 1.0], [2.0, 3.0]]

x[0][0] = 10; // ok
x[0] = 10; // error

An array without a type parameter can contain any type (i.e. falls back to Array<any>).

Set

Similar to an array, a set takes a single optional type parameter (which can itself be a set):

var s Set<Number> = new Set();

x.add(10); // ok
x.add(false); // error

Map

A map takes two type parameters:

var m Map<Int, String> = new Map();

m.set(0, "first entry"); // ok
m.set("first entry", 0); // error

Object

Unlike vanilla JS the Object class is not considered to be the parent class of nearly every instance. Instead it is solely used as a dict-like container type, mapping string keys to arbitrary values.


Dict-like type parameters are used to specify the types of these values:

var obj Object<key1: String, key2: Array<String>> = {
key1: "value",
key2: ["a", "b", "c"],
}

If the Object entry names are not known, but only the common value type, then a single unnamed type parameter can be used for the value type:

var obj Object<String> = {};
obj["key1"] = "value"; // ok
obj["key2"] = ["a", "b", "c"]; // Error: expected String, got Array<String>

An Object without type parameters falls back to Object<any>.

Tuple

Tuples are fixed length arrays. Each tuple entry can have a different type. Tuple type parameters are not optional, and at least 2 must be specified.

var pair Tuple<String, Object, Int> = new Tuple("name", {}, 0);

When looping over a Map<KeyType, ValueType> using for (let x of ...), the type of x is Tuple<KeyType, ValueType>.

Promise<ResolveType>

ResolveType can be void.

Event<TargetType>

TargetType is typically HTMLElement.

IDBRequest<ResultType>

function<ArgType1, ..., ReturnType>

ReturnType can be void.

class<ArgType1, ..., InstanceType>

The ArgTypes are the arguments of the constructor. The InstanceType is a class or an interface.
