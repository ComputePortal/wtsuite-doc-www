Overview of tools

This page summarizes the command-line transpiler tools included in wtsuite.

wt-site

wt-site builds an entire static website using a json config file as input.


Usage:

> wt-site site.json . --auto-link

An example config file can be found here.

wt-search-index

wt-search-index indexes pages mentioned in the site config file. Additional settings can be used to refine the indexation. The index is stored in a json file, which can be used by a builtin javascript class to provide client-side search.


Usage:

> wt-search-index site.json search-index.json --auto-link

wt-template

wt-template transpiles a single template file, and produces a single html file, with all styles and scripts included in the html file. This is the easiest way to get started with the template engine.


Usage:

> wt-template index.wtt -o index.html --auto-link

Once you start creating websites with many pages, then consider using the wt-site tool instead.

wt-template-syntax-tree

wt-template-syntax-tree dumps the syntax tree of template files. This is used by wtsuite developers to debug their modifications of the parsers.

wt-script

wt-script transpiles and bundles typed javascript files. It needs an entry point to get started (other files are found using import statements).


Usage:

> wt-script main.wts -o main.js --target nodejs --executable

wt-script-grapher

wt-script-grapher creates graphviz dot files representing the structure of your wtsuite script programs. Useful when analyzing larger projects.

wt-script-refactor

wt-script-refactor renames script classes and packages. Related file names are also renamed. This is safer than using search-and-replace. Comments are left untouched.

wt-script-syntax-tree

wt-script-syntax-tree dumps the syntax tree of script files. This is used by wtsuite developers to debug their modifications of the parsers.

wt-svg-minify

wt-svg-minify is used to minify svg files created by inkscape.

wt-glsl

wt-glsl transpiles and bundles shader files. It needs an entry point to get started (other files are found using import statements).


Usage:

> wt-glsl main.glsl -o main.shader --target fragment

wt-glsl-syntax-tree

wt-glsl-syntax-tree dumps the syntax tree of shader files. This is used by wtsuite developers to debug their modifications of the parsers.

wt-style

wt-style transpiles a template script file into a css file. The input file must have an exported style variable called main.

Usage:

> wt-style ./style/main.wtt -o ./style.css
