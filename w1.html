WebGLProgram macro

Shader code can be placed in separate files and transpiled separately from javascript. The WebGLProgram macro provides convenient access to this transpilation from within javascript.

Usage

The literal paths of the vertex and fragment shaders must be passed into the WebGLProgram macro:

canvas := cast(document.getElementById("canvas"), HTMLCanvasElement);
gl := canvas.getContext("webgl");
program := new WebGLProgram(gl, "./vertex-shader.glsl", "./fragment-shader.glsl");

The macro takes care of the transpilation, compilation and linkage. The returned value is of WebGLProgram type.

Constants

Constants exported from vertex or fragment shaders can be overwritten via the WebGLProgram macro:

program := new WebGLProgram(gl,
"./vertex-shader.glsl", {maxIter: 1000},
"./fragment-shader.glsl", {}
);

In this example maxIter is the name of the exported constant, and it must be an int (otherwise a transpiler error is thrown).
