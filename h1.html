Tags

Tags in template script are indentation based. Specifying attributes is similar to passing arguments into functions.

Minimal html page

wtsuite's template script uses indentations to nest tags. The code needed for a minimal page is:

// !doctype is automatically added
html
head
body

Attributes

Attributes are passed into tags similar to how keyword arguments are passed into functions:

body
div(id="content")
input(type="text", placeholder="...")

Text

Text nodes are literal strings, and, like regular tags, the indentation is used to determine under which tag to place them:

div
a(href="https://computeportal.com")
"Click here"

Literal strings can be placed inline with some tags:

div
a(href="https://computeportal.com") "Click here"

Multiple text tags can be placed inline, each new tag being nested in the previous tag. You can then use the < op to move back up a level:

div
p "normal text " b "bold text" < " normal text"

Note how the spaces before and after the b tag are preserved.


Single quotes ', double quotes ", and backticks ` can be used to specify literal strings. Each type of quotes escapes the other types of quotes.
