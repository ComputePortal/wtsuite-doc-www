Enum types

Enums are collections of predefined constants. Like java, enums in wtsuite's typed javascript are syntactic sugar for classes with some automatically generated static members.

Simple example

Enums are defined as follows:

enum ErrorCode { // implicitely extends Int
NO_ERROR, // 0
SYNTAX_ERROR, // 1
GENERIC_ERROR // 2
}

Enums are class-like types, and can be used as follows:

function printErrorMessage(code ErrorCode) {
console.log(ErrorCode.key(code)); // prints 'NO_ERROR', 'SYNTAX_ERROR' or 'GENERIC_ERROR'
}

printErrorMessage(ErrorCode.GENERIC_ERROR);

Instances of enums can still be used as instances of their parent types:

function addInts(a Int, b Int) {
return a+b;
}

addInts(ErrorCode.NO_ERROR, ErrorCode.GENERIC_ERROR) == ErrorCode.GENERIC_ERROR; // true

Advanced example

This example is similar to the example in Oracle's java documentation on enums (this way you can immediately see the difference between java and wtsuite's typed javascript):

class Planet {
mass Number;
radius Number;

constructor(m Number, r Number) {
this.mass = m; // in kg
this.radius = r; // in m
}
}

enum SSPlanet extends Planet {
MERCURY = new Planet(3.303e+23, 2.4397e6),
VENUS   = new Planet(4.869e+24, 6.0518e6),
EARTH   = new Planet(5.976e+24, 6.3781e6),
MARS    = new Planet(6.421e+23, 3.3972e6),
JUPITER = new Planet(1.900e+27, 7.1492e7),
SATURN  = new Planet(5.688e+26, 6.0268e7),
URANUS  = new Planet(8.686e+25, 2.5559e7),
NEPTUNE = new Planet(1.024e+26, 2.4746e7)
}

Details

Enum members are separated by commas. Each enum member has a name and a value.


Implicit values

The value is optional in some cases. If it is left out for each member, then the enum implicitely extends Int, and each member is assigned a unique integer value, starting at 0 and incremented by 1 for each following member.


There is no auto-incrementing if some, but not all, values are specified.


If the first member value is a literal string which exactly matches the name of that member, then unspecified remaining values are implicit strings with the same value as the names of those other members.


All enum values must be specified in all other cases.


Implicit parent class

If no parent class is specified, and if all values are implicit integers, or literal integers, then the enum implicitely extends Int.


The String class is similarly implicitely extended if all values are implicit or literal strings.


A parent class must be specified in all other cases.


Static member functions

  • <EnumType>.values() Array<EnumType> : returns array of enum members.
  • <EnumType>.value(key String) EnumType : returns enum member which has same name as key . Returns null if not found.
  • <EnumType>.keys() Array<String> : returns array of enum members names.
  • <EnumType>.key(x EnumType) String : returns name of the enum value.

Enum members can't have the same name as one of these functions. Using capitalized names for enum members avoids any such naming conflicts.
